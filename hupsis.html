Mumpsis - an information system for events and an automatic plan for servings

This is practice work I made for Data analytics and knowledge-based management training of Opiframe. I decided to focus my work on building a database and maintaining it through the website.


The purpose of the work was to develop an information system called Hupsis and build an add-on to it. The next step is to build the storage of the plans, after which I will make the suggestions of the plans more flexible with the help of machine learning.

The languages and technologies used in this work

  • CSS3

  • ECMAScript
  • PostgreSQL 9.2.24
  • phpPgAdmin 5.6
  • Brackets


The codes I have used can be found in my GitHub-accounts Hupsis-repository: GitHub.

Background for the work

Finnish Red Cross first aid volunteers use an information system called Hupsis to manage Red Crosses first aid - medical services nationwide. With Hupsis we can inform volunteers about events and be sure we have volunteers enough. There must be a first aid plan and risk assessment for every event. At the time we use Word and Excel templates to make those plans, but by automating those actions with Hupsis-add-on, plans and assessments will be uniform quality and it would save time for volunteers.
 

Because templates are for our own use and Hupsis is not an open system I will make this add-on as catering of events, first aid plan is plan for catering and risk assessment is amounts of served foods. If the add-on is useful it can be modified to Hupsis with necessary changes so it can be used as originally designed.

Functionality

If you wish to see events from database, create automatic plan for servings or delete events you can continue from the other link.

You can also print/save this plan in pdf format. Currently, edited plans cannot be saved, but this functionality is coming in the future.
